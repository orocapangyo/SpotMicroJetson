# 3개월 4족로봇 스터디 계획서

## 주제

**BLDC 기반 4족로봇 액추에이터/단일 다리 프로토타입 제작 + 시뮬레이션 기반 보행 학습**

최종 목표는 Unitree A2급 4족로봇과 같은 고성능 4족 플랫폼을 장기적으로 개발하는 것이다. 3개월 스터디에서는 전체 4족로봇을 무리하게 제작하지 않고, 실제 하드웨어에서는 핵심 요소인 **BLDC 감속 액추에이터와 단일 다리 구조**를 검증하고, 4족 보행은 **Isaac Sim / Isaac Lab / MuJoCo 등 시뮬레이션 환경**에서 학습 및 분석한다.

---

## 1. 프로젝트 방향 요약

### 1.1 최종 장기 목표

- Unitree A2급 4족로봇 개발을 위한 기초 기술 확보
- 고출력 BLDC 액추에이터 설계 능력 확보
- 4족로봇 다리 구조, 관절 토크, 감속기, 엔코더, FOC 제어 이해
- 시뮬레이션 기반 보행 학습 및 gait control 이해
- 장기적으로 실물 12축 4족로봇 제작으로 확장

### 1.2 3개월 스터디 목표

3개월 안에 전체 4족로봇을 완성하는 것이 아니라, 다음 2개 축으로 진행한다.

#### 하드웨어 트랙

- 보유 중인 **80mm Kv80 BLDC 모터 + ODrive S1** 기반 1축 액추에이터 제작
- 감속기 또는 벨트 감속 구조 적용
- 엔코더 기반 위치/속도/토크 제어 실험
- 레버암과 무게추를 이용한 출력축 토크 테스트
- 가능하면 1자유도 → 2자유도 → 3자유도 단일 다리 구조까지 확장

#### 시뮬레이션 트랙

- Isaac Lab / MuJoCo / PyBullet 중 하나를 사용해 4족로봇 보행 환경 실행
- 기존 quadruped 예제 로봇 기반으로 보행 policy 실행 또는 학습
- observation, action, reward, joint torque, gait 구조 분석
- 실제 액추에이터 설계와 시뮬레이션 요구 토크/속도 비교

## 2. 왜 전체 4족로봇이 아니라 액추에이터와 단일 다리부터 하는가

4족로봇의 성능은 프레임보다 **관절 액추에이터**에서 대부분 결정된다. A2급 로봇을 만들려면 12개의 고성능 관절이 필요하다. 하지만 1축도 충분히 검증되지 않은 상태에서 12축으로 확장하면 비용과 디버깅 난이도가 급격히 증가한다.

따라서 이번 스터디에서는 다음 순서를 따른다.

1. 1축 BLDC 액추에이터 검증
2. 감속기/출력축/베어링/엔코더 구조 검증
3. 토크/속도/발열 데이터 확보
4. 단일 다리 구조로 확장
5. 보행은 시뮬레이션에서 먼저 학습
6. 이후 실물 4족로봇으로 확장

이 방식은 실패 원인을 분리하기 쉽고, 장기적으로 A2급 로봇 개발에 필요한 핵심 데이터를 얻을 수 있다.

## 5. 핵심 계산 기준

### 5.1 Kv와 Kt 관계

Kv 80 rpm/V 모터의 토크상수 Kt는 대략 다음과 같이 추정할 수 있다.

```text
Kt ≈ 60 / (2π × Kv)
```

Kv = 80 rpm/V일 때,

```text
Kt ≈ 0.119 Nm/A
```

즉, 이론적으로 모터축 토크는 다음과 같다.

| 전류 | 모터축 토크 이론값 |
|---:|---:|
| 10A | 약 1.2Nm |
| 20A | 약 2.4Nm |
| 40A | 약 4.8Nm |
| 80A | 약 9.5Nm |

### 5.2 감속 후 출력축 토크

출력축 토크는 다음과 같이 계산한다.

```text
출력축 토크 ≈ 모터축 토크 × 감속비 × 감속기 효율
```

감속기 효율을 80%로 가정하면:

| 감속비 | 40A 기준 | 80A 기준 |
|---:|---:|---:|
| 10:1 | 약 38Nm | 약 76Nm |
| 15:1 | 약 57Nm | 약 114Nm |
| 20:1 | 약 76Nm | 약 152Nm |

주의할 점은 위 값은 이론값이며 실제 성능은 모터 발열, 드라이버 전류 제한, 감속기 강도, 베어링 구조, 냉각 상태에 따라 크게 달라진다.

### 5.3 레버암 토크 테스트

실제 출력축 토크는 레버암과 무게추로 검증한다.

```text
토크 = 무게 × 9.81 × 레버 길이
```

예시:

| 무게 | 레버 길이 | 필요 토크 |
|---:|---:|---:|
| 5kg | 0.2m | 약 9.8Nm |
| 10kg | 0.2m | 약 19.6Nm |
| 15kg | 0.2m | 약 29.4Nm |
| 20kg | 0.2m | 약 39.2Nm |

## 8. 주의사항

### 8.1 하드웨어 안전

- 48V 이상 전원 사용 시 감전 및 쇼트 주의
- 고전류 배선에는 충분한 굵기의 케이블 사용
- 퓨즈 또는 차단기 적용 권장
- 실험 중 비상정지 스위치 준비
- 모터가 갑자기 회전할 수 있으므로 손이나 케이블이 말려들지 않게 할 것
- 레버암 토크 테스트 시 무게추 낙하 방지

### 8.2 ODrive 설정 주의

- 처음에는 전류 제한을 낮게 설정
- 캘리브레이션 중 모터가 튈 수 있음
- 엔코더 방향이 틀리면 제어가 불안정해짐
- 위치제어 gain을 천천히 올릴 것
- 고전류 테스트 전 냉각 상태 확인

### 8.3 기구 설계 주의

- 모터축에 직접 외부 하중을 걸지 말 것
- 출력축은 별도 베어링으로 지지
- 감속기와 출력축 정렬 중요
- 벨트 감속 사용 시 장력 조절 구조 필요
- 백래시와 유격을 반드시 기록

### 8.4 시뮬레이션 주의

- 처음부터 커스텀 로봇으로 학습하지 말 것
- 기존 quadruped 예제를 먼저 실행
- 모델 질량, 관성, 충돌체 설정 오류가 학습 실패의 주요 원인
- reward를 한 번에 크게 수정하지 말 것
- 보행 영상만 보지 말고 관절 토크/속도 로그를 확인할 것

## 13. 결론

이번 3개월 스터디의 방향은 **전체 4족로봇 제작**이 아니라 **핵심 기술 검증**이다.

실물에서는 BLDC 액추에이터와 단일 다리 구조를 통해 하드웨어 감각을 확보하고, 시뮬레이션에서는 4족 보행학습을 진행하여 제어 알고리즘과 관절 요구사항을 이해한다.

이 두 결과를 연결하면, 이후 실제 12축 4족로봇을 설계할 때 필요한 다음 기준을 얻을 수 있다.

- 필요한 관절 토크
- 필요한 관절 속도
- 적절한 감속비
- 모터/드라이버 전류 요구량
- 발열 한계
- 다리 링크 길이
- 보행 제어 구조
- 시뮬레이션과 실물 간 차이

따라서 이번 스터디는 A2급 4족로봇을 향한 첫 번째 실전 단계로 정의한다.
